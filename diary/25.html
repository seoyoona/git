git 로컬 작업 순서

  • 폴더별 최초 한 번 깃 초기화
    • git init
  • 스테이지에 올림
    • git add .
  • pc별 최초 한 번 사용자 등록
    • git config --global user.email ""
    • git config --global user.name ""
  • 폴더별 최초 한 번 브랜치명 변경
    • git branch -M main

git 원격 작업 순서

  • 폴더별 최초 한 번
    • create repository : 버트 클릭 후 정보 입력해 저장소 생성
  • 생산된 repository 주소 복사
  • git add . : 편집파일 스테이지로 올림
  • git commit -m "commit message" : 버전 생성
  • git push origin main : 원격 저장소에 코드 올림

CSS UI

em / rem

  • em : 대문자 M의 너비를 기준으로 한 단위 = 약 16px (부모기준)
  • rem : 대문자 M의 너비를 기준으로 한 단위 = 약 16px (root 기준)

vh / vw

  • vh (viewport height) : 웹브라우저 높이 기준
  • vw (viewport width) : 웹브라우저 너비 기준

+ 가변 크기 구하는 공식

= 자식크기/부모크기 x 100

+ vh / vw 크기 구하는 공식

= (폰트크기 x 화면높이 or 화면너비) / 100

percent / vw

  • percent : 부모기준
  • vw : 브라우저 너비 기준

width : 50%width : 80%

width : 50vw width : 80vw

MediaQuery

= 화면 크기에 따른 각각의 속성 값을 지정하여, 여러가지 화면을 구성하는 기술을 말합니다.

@media only screen and (조건문) {실행문}

  • @media : 미디어 쿼리가 시작됨
  • only : 미디어 쿼리 구문을 해석하라는 명령어 (생략가능)
  • all : 미디어 쿼리를 해석해야 할 대상을 나타냄 (생략가능)
    • all : 모든 미디어 유형에서 사용할 css 정의
    • print : 인쇄장치에서 사용할 css 정의
    • screen : 컴퓨터 스크린에서 사용할 css 정의
  • (조건문) : 해당 크기 설정 가능
  • {실행문} : 조건에 따른 실행 설정

+ Size

  • 화면 너비 0~1280px : pc
  • 화면 너비 0~1024px : pc
  • 화면 너비 0~960px : 노트북
  • 화면 너비 0~768px : 테블릿
  • 화면 너비 0~576px : 모바일
Ex.  @media   screen   and   (max-width: 1280px)   {body {background: lightsteelblue;} h1 {transform: translateX(100px);}}
